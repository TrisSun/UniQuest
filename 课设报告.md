## 一、需求分析

灵感来源于游戏与小说里常见的"冒险者公会"。本系统把它做成一个面向**江苏省高校学生**的委托互助平台:同学之间可以把自己需要帮忙的事情("跑腿带饭""帮忙写海报""一起做实验"等)作为**委托**发布出来,其他同学可以**接取**这些委托,完成后通过**经验值**升级,形成一个有成长感的校园互助社区。

为什么要做这样一个系统:

- 校园里经常有零散的互助需求,缺少一个集中、有趣的平台来撮合;
- 用"等级、经验"的游戏化机制鼓励大家积极完成委托,比单纯的留言板更有动力;
- 限定"仅江苏省高校学生可用",保证用户群体真实、范围可控。

为什么采用 C++:

- C++ 支持**面向对象**编程,可以把"冒险者""委托""公会"分别抽象成类,结构清晰、易于维护和扩展;
- C++ 标准库的文件流(`fstream`)能方便地把数据保存到文件,实现数据持久化;
- 是本学期课程所学语言,适合作为课程设计来综合运用所学知识。

## 二、程序的主要功能

1. **注册功能**:输入用户名、学校、密码注册账号。系统会校验学校是否为江苏省高校,只有江苏高校学生才能注册成功。
2. **登录功能**:输入用户名和密码登录,密码错误或用户不存在会给出提示。
3. **发布委托**:登录后可以发布委托,填写标题、描述、奖励经验和难度。
4. **接取委托**:浏览委托公告栏,接取处于"待接取"状态的委托(不能接取自己发布的委托)。
5. **提交完成委托**:接取者完成委托后提交,委托变为"已完成",接取者获得对应经验,经验足够时自动升级。
6. **浏览所有委托**:查看公告栏里所有委托的编号、状态、难度、奖励、标题和发布者。
7. **查看我的信息**:查看自己的等级、经验、所在学校,以及自己发布和接取的委托。
8. **数据持久化**:所有用户和委托数据都保存在文本文件中,程序重新打开后数据依然存在。

## 六、模块分析

1. **注册模块 `registerUser`**:提示用户输入用户名、学校、密码。先检查用户名是否重复,再调用 `isJiangsuUniversity` 校验学校,通过后把新用户加入 `users` 并保存文件。
2. **登录模块 `login`**:输入用户名、密码,调用 `findUser` 查找用户并比对密码,成功则记录 `currentUserIndex`。
3. **发布委托模块 `publishQuest`**:登录用户输入标题、描述、奖励经验、难度,用 `nextQuestId` 作为新编号创建 `Quest` 对象加入 `quests` 并保存。
4. **接取委托模块 `takeQuest`**:先显示公告栏,用户输入编号;校验委托是否"待接取"、是否为自己发布,通过则把状态改为"进行中"并记录接取者。
5. **完成委托模块 `completeQuest`**:列出当前用户"进行中"的委托,输入编号后把状态改为"已完成",调用 `gainExp` 给接取者加经验并可能升级。
6. **显示模块 `showAllQuests` / `showMyInfo`**:分别输出公告栏全部委托,以及当前用户的等级、经验与相关委托。
7. **文件模块 `load*/save*`**:负责把内存中的对象与文本文件互相转换,实现数据持久化。

## 七、比较有特色的算法

### 1. 完成委托后的等级计算(经验阈值连升级算法)

每一级升到下一级需要 `level * 100` 点经验。完成委托得到经验后,用一个 `while` 循环不断判断:只要当前经验够升级,就扣掉本级所需经验、等级加一,直到经验不足以再升级为止。这样一次获得大量经验时可以**连升多级**,且多余经验会保留到下一级。

```cpp
int Adventurer::expToNext() const {
    return level * 100;          // 升级公式
}

void Adventurer::gainExp(int amount) {
    if (amount > 0) exp += amount;
    while (exp >= expToNext()) { // 经验够就一直升级
        exp -= expToNext();
        level++;
    }
}
```

特色:用简单的循环实现了"经验溢出连升级",逻辑清晰、易于调整难度曲线(只要改升级公式即可)。

### 2. 委托状态机 + 权限校验

每个委托有"待接取 → 进行中 → 已完成"三个状态,程序通过校验保证状态只能合法流转:

- 只有"待接取"的委托才能被接取;
- 不能接取自己发布的委托;
- 只有接取者本人、且委托处于"进行中"时才能提交完成。

```cpp
// 接取委托时的校验(节选)
if (quests[i].getStatus() != QUEST_WAITING) {
    cout << "该委托已被接取或已完成,无法接取!" << endl;
    return;
}
if (quests[i].getPublisher() == me) {
    cout << "不能接取自己发布的委托!" << endl;
    return;
}
quests[i].setTaker(me);
quests[i].setStatus(QUEST_TAKEN);
```

特色:用一个整型状态 + 几处条件判断,实现了一个小型"状态机",保证了业务流程的正确性和数据的合法性。

## 八、存在的不足与对策、编程体会

**不足与对策:**

1. 密码以明文保存在文件中,不安全。对策:后续可加入简单的加密(如异或、哈希)。
2. 查找用户和委托使用线性遍历,数据量大时效率较低。对策:数据量大时可改用 `map` 按用户名/编号建立索引。
3. 标题、描述中如果输入竖线 `|`,会破坏文件格式。对策:输入时提示不要使用该符号,或改用更健壮的转义/序列化方式。
4. 目前没有管理员审核、没有委托搜索分类。对策:可扩展管理员角色、关键字搜索、积分排行榜等功能。

**编程体会:**

通过本次课程设计,我把"冒险者""委托""公会"抽象成类,真正体会到面向对象"封装"带来的清晰结构:每个类只负责自己的数据和行为,`Guild` 负责把它们组织起来。文件读写让我理解了数据持久化的意义。整个过程中也学会了把功能拆分成小函数、用 `vector` 管理对象集合,以及处理用户的非法输入。

## 九、使用说明

### 编译

方法一(命令行,需安装 g++):

```bash
g++ -std=c++11 -o guild main.cpp Adventurer.cpp Quest.cpp Guild.cpp
```

或在该目录直接执行 `make`。

方法二(IDE):在 Dev-C++ / Visual Studio 中新建项目,把 `main.cpp`、`Adventurer.cpp/.h`、`Quest.cpp/.h`、`Guild.cpp/.h` 全部加入项目后编译运行。

### 运行步骤

1. 运行程序,出现欢迎界面和"请先登录"菜单。
2. 选 `1` 注册:输入用户名,**学校必须填江苏省高校**(如:南京航空航天大学、南京大学、东南大学、苏州大学等),再设置密码。
3. 选 `2` 登录:输入刚才的用户名和密码。
4. 登录后在主菜单中:
   - `1` 浏览所有委托;
   - `2` 发布委托(填写标题、描述、奖励经验、难度);
   - `3` 接取别人的委托;
   - `4` 提交自己完成的委托,获得经验、可能升级;
   - `5` 查看我的等级、经验和委托;
   - `6` 退出登录,`0` 退出系统。
5. 数据自动保存在程序目录下的 `users.txt` 和 `quests.txt`,下次打开依然存在。
